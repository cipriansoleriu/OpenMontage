# Stage: character_bible

Goal: produce, for each character, a reference sheet + a FIXED identity block reused verbatim
in every later generation. This is the foundation of consistency.

Do:
1. If the user supplied real photos, use them as references. Otherwise generate a sheet with
   `nano_banana_pro`: one front headshot, one 3/4 portrait, one full-body, 1-2 action poses —
   all the SAME person (pass previous outputs back as references to hold identity).
2. Write a 50-80 word identity block (face, hair, build, wardrobe, signature details). Store it.
3. Checkpoint the character (sheet paths + identity block) to the project + global registry.
4. PAUSE at the `character` gate: show the sheet, get explicit approval before spending on training.
Never proceed to animation from here — identity must be trained/locked first.
